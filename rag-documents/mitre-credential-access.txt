MITRE ATT&CK Framework - Credential Access (TA0006)
=====================================================

The Credential Access tactic covers techniques adversaries use to steal account credentials such as usernames and passwords. Stolen credentials may give attackers access to systems and enable persistent access, privilege escalation, and lateral movement.

---

T1110 - Brute Force
---------------------
Adversaries attempt to gain access to accounts by systematically guessing passwords.

Sub-techniques:
- T1110.001 - Password Guessing: Trying common or dictionary-based passwords.
- T1110.002 - Password Cracking: Offline cracking of captured password hashes using tools like Hashcat or John the Ripper with GPU acceleration.
- T1110.003 - Password Spraying: Trying a small number of commonly used passwords (e.g., "Winter2024!") against many accounts simultaneously to avoid lockouts.
- T1110.004 - Credential Stuffing: Using username/password pairs from prior data breaches to attempt logins on other services, exploiting password reuse.

Detection: Multiple failed login attempts from same source, successful login after many failures, logins from unusual geolocations, anomalous authentication velocity.

Mitigation: Account lockout policies (balanced to prevent DoS), MFA, CAPTCHA on login pages, password managers to prevent reuse, monitoring via SIEM.

Procedure Examples:
- Iranian APT groups (APT33) used password spraying against oil & gas companies.
- Credential stuffing attacks are responsible for millions of account takeovers annually.
- OilRig used brute force against VPN portals.

T1555 - Credentials from Password Stores
------------------------------------------
Adversaries search for and extract saved credentials from password storage locations.

Sub-techniques:
- T1555.001 - Keychain: MacOS keychain stores Wi-Fi passwords, website credentials, and certificates.
- T1555.002 - Securityd Memory: Extracting credentials from macOS securityd process memory.
- T1555.003 - Credentials from Web Browsers: Browsers store credentials in encrypted databases (Chrome: Login Data, Firefox: key4.db/logins.json). Tools like LaZagne can extract these.
- T1555.004 - Windows Credential Manager: Windows stores credentials for network resources, websites.
- T1555.005 - Password Managers: Targeting password manager applications if unlocked.

Tools: LaZagne, Mimikatz, Browser credential dumpers.

Detection: Monitor access to credential storage files, suspicious process access to browser profile directories, use of known credential dumping utilities.

Mitigation: Endpoint detection and response (EDR), application whitelisting, credential encryption, encourage enterprise password manager use.

T1003 - OS Credential Dumping
-------------------------------
Adversaries attempt to dump credentials to obtain account login information for use in lateral movement.

Sub-techniques:
- T1003.001 - LSASS Memory: Dumping Local Security Authority Subsystem Service (LSASS) memory contains credentials in Windows. Tools: Mimikatz (sekurlsa::logonpasswords), Procdump, Task Manager.
- T1003.002 - Security Account Manager (SAM): Windows SAM database contains local account password hashes. Can be read when SYSTEM-level access is obtained. Tools: reg.exe, Impacket.
- T1003.003 - NTDS: Active Directory database (NTDS.DIT) contains domain credentials. Volume Shadow Copies can be used to access locked files.
- T1003.004 - LSA Secrets: Registry-stored credentials for services, autologon, VPN, and scheduled tasks.
- T1003.005 - Cached Domain Credentials: Windows caches domain credentials locally (MS-Cache2 / DCC2 hashes).
- T1003.006 - DCSync: Mimikatz lsadump::dcsync replicates Active Directory to extract password hashes without touching LSASS.
- T1003.007 - Proc Filesystem (Linux): /proc/<PID>/mem and /proc/<PID>/maps to dump process memory.
- T1003.008 - /etc/passwd and /etc/shadow: Linux shadow file contains hashed passwords.

Detection: Monitor LSASS process memory access, use Windows Credential Guard, alert on known dumping tool hashes/behavior, SIEM rules for suspicious registry access to SAM/SECURITY hives.

Mitigation: Windows Credential Guard (prevents LSASS memory access), Protected Users security group, Privileged Access Workstations (PAW), disable debug privileges for non-admin users.

Procedure Examples:
- Almost every major ransomware group (Conti, LockBit, BlackCat) uses Mimikatz for credential dumping.
- APT29 (Cozy Bear) used DCSync against Active Directory.
- HAFNIUM dumped LSASS to harvest credentials post-Exchange exploitation.

T1558 - Steal or Forge Kerberos Tickets
-----------------------------------------
Kerberos is the default authentication protocol in Active Directory environments. Adversaries abuse it to forge tickets.

Sub-techniques:
- T1558.001 - Golden Ticket: Forging a Ticket Granting Ticket (TGT) using the KRBTGT account hash, giving persistent admin access to any service. Requires domain controller compromise.
- T1558.002 - Silver Ticket: Forging a service ticket (TGS) for a specific service using the service account hash.
- T1558.003 - Kerberoasting: Requesting Kerberos service tickets for service accounts with SPNs, then cracking the ticket offline to reveal the service account password.
- T1558.004 - AS-REP Roasting: Targeting accounts with Kerberos pre-authentication disabled, allowing offline cracking of authentication responses.

Tools: Mimikatz, Impacket (GetUserSPNs.py, GetNPUsers.py), Rubeus.

Detection: Abnormal Kerberos ticket lifetimes, tickets for services the user never normally accesses, Event IDs 4768/4769/4771 anomalies.

Mitigation: Strong service account passwords (25+ chars), Microsoft ATA/Defender for Identity, KRBTGT password rotation after breach, disable accounts without pre-auth requirement.

T1056 - Input Capture
-----------------------
Adversaries use keyloggers and other input capture mechanisms to collect user-entered data including credentials.

Sub-techniques:
- T1056.001 - Keylogging: Software or hardware keyloggers record keystrokes. Examples: commercial RATs with keylogging features, custom drivers.
- T1056.002 - GUI Input Capture: Fake windows or UI elements that capture credentials entered by users.
- T1056.003 - Web Portal Capture: Injecting code into web portals to capture credentials at submission.
- T1056.004 - Credential API Hooking: Hooking Windows API calls (CredEnumerateA, WinVerifyTrust) to intercept credentials.

Detection: Behavioral monitoring for API hook injection, suspicious DLL injection, process access to credential-related APIs.

Mitigation: EDR solutions, application sandboxing, user training to verify login page authenticity.

T1539 - Steal Web Session Cookie
----------------------------------
Adversaries steal session cookies from authenticated web sessions to bypass authentication and MFA.

Techniques:
- Adversary-in-the-Middle (AiTM) phishing proxies capture session tokens in real-time (EvilGinx2, Modlishka, Muraena).
- Malware extracting cookies from browser databases.
- Cross-site scripting (XSS) to steal cookies.

Recent campaigns:
- Microsoft observed AiTM phishing campaigns targeting over 10,000 organizations in 2022, bypassing MFA by stealing session tokens.

Detection: Anomalous session token usage (different device/IP/browser), impossible travel alerts.

Mitigation: Token binding, continuous access evaluation, device compliance requirements, Conditional Access with device fingerprinting.

T1552 - Unsecured Credentials
-------------------------------
Adversaries search compromised systems for credentials stored insecurely.

Sub-techniques:
- T1552.001 - Credentials in Files: Config files, scripts, source code containing hardcoded passwords.
- T1552.002 - Credentials in Registry: Windows Registry entries storing plaintext or encoded credentials.
- T1552.003 - Bash History: Unix shell history files often contain commands with passwords.
- T1552.004 - Private Keys: SSH private keys, code signing certificates stored without passphrase.
- T1552.005 - Cloud Instance Metadata API: AWS/Azure/GCP metadata services providing IAM credentials to EC2/VM instances. If SSRF exists, attackers can steal these.
- T1552.006 - Group Policy Preferences: Legacy GPP XML files stored SYSVOL contained encrypted credentials (AES key was publicly disclosed).

Detection: File system monitoring, baseline deviation on sensitive file access, monitoring for metadata API calls.

Mitigation: Secret management platforms (HashiCorp Vault, AWS Secrets Manager), pre-commit hooks to prevent credential commits, credential scanning in CI/CD.
